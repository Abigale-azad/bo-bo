jrsy 云端同步

让你的聊天记录永不丢失，换设备、清缓存都能自动恢复。

快速开始

只需要 3 步，大约 5 分钟。

1 创建 Firebase 项目

访问 Firebase 控制台

  1. 点击 "添加项目"（或 "Create a project"）
  2. 输入项目名称（如 "jrsy-sync"），点击继续
  3. 关闭 Google Analytics（不需要），点击"创建项目"
  4. 等待项目创建完成，点击"继续"

2 获取配置信息

  1. 在项目概览页，点击 "Web"（</>）图标 添加 Web 应用
  2. 输入应用名称（如 "jrsy"），点击"注册应用"
  3. 复制弹出的 firebaseConfig 内容（类似下面这样）： 
    {
  apiKey: "AIzaSyDxxxxxxxxxxxxxxxxxxxxxx",
  authDomain: "jrsy-sync.firebaseapp.com",
  projectId: "jrsy-sync",
  storageBucket: "jrsy-sync.appspot.com",
  messagingSenderId: "123456789",
  appId: "1:123456789:web:abc123def456"
}

3 启用 Firestore 数据库

  1. 在左侧菜单点击 "Firestore Database"
  2. 点击 "创建数据库"
  3. 选择 "以测试模式启动"（开发阶段推荐）
  4. 选择数据库位置（默认即可），点击"启用"
注意：测试模式允许任何人读取你的数据，正式使用前请务必修改安全规则。

4 在 jrsy 中配置

  1. 打开 jrsy 应用，进入 "我" → 设置 → 云同步
  2. 点击 "配置云同步"
  3. 填入从 Firebase 复制的 apiKey 和 projectId
  4. 点击 "保存并启用"
配置成功后，右上角会显示绿色"已同步"提示。每次数据变动都会自动保存到云端。

部署你的网页

部署到 GitHub Pages（推荐）

  1. 在 GitHub 上创建一个新的公开仓库（如 jrsy-cloud）
  2. 将 dist/ 文件夹里的所有文件上传到仓库
  3. 进入仓库 Settings → Pages
  4. Source 选择 "Deploy from a branch"，分支选 main
  5. 点击 Save，等待几分钟，访问 https://你的用户名.github.io/jrsy-cloud/

Q&A

Q: 免费够用吗？
Firebase 免费额度：存储 1GB，每天 5 万次读取 / 2 万次写入。个人使用完全够用。

Q: 数据安全吗？
你的数据存储在你自己的 Firebase 项目中，只有你能访问。
测试模式 30 天后会失效，请在 Firebase 控制台 → Firestore → 规则 中设置安全规则：
allow read, write: if true;（保持个人使用开放即可）

Q: 怎么从旧设备迁移数据？
在旧设备上配置好云同步后，数据会自动上传。在新设备上用同样的激活码登录，数据会自动恢复。

Q: 数据同步频率？
每次数据保存后 3 秒自动上传（防抖合并），避免频繁请求。
